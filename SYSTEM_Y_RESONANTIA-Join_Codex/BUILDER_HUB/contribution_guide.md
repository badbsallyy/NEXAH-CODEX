## 🅁 Contribution Types · Beitragsformen

| Type / Typ                          | Description / Beschreibung                                                                                                                                    |
| :---------------------------------- | :------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| 🎨 **Visual Contribution**          | Upload a `.png`, `.jpg`, or `.svg` visual to illustrate a Codex field. <br> Lade ein Visual hoch, das ein Resonanzfeld oder Konzept darstellt.                |
| 🎵 **Sound Field / Audio**          | Contribute `.wav` or `.mp3` soundscapes derived from frequency ratios or Codex themes. <br> Erstelle Klangfelder oder tonale Resonanzkompositionen.           |
| 🔢 **Mathematical / Symbolic Work** | Add `.md` or `.ipynb` files containing symbolic derivations or equations. <br> Füge mathematische Herleitungen oder Formeln als Markdown oder Notebook hinzu. |
| 🧠 **Code / Simulation**            | Submit small scripts in `.py`, `.js`, or `.pde` that visualize Codex data. <br> Reiche Code-Snippets oder interaktive Simulationen ein.                       |
| 📜 **Text / Myth / Narrative**      | Write short texts, myths, or conceptual reflections in `.md`. <br> Schreibe kurze Texte oder symbolische Erzählungen.                                         |

## 🅄 Naming Conventions · Benennung

**EN:**
Use clear and lowercase filenames with underscores:
`module_name_contributor_type.ext`

**DE:**
Verwende klare, kleingeschriebene Dateinamen mit Unterstrichen:
`modul_name_beitragstyp.ext`

**Examples / Beispiele:**

* `quantum_breach_visual_thomas.png`
* `lambda_disc_equation_note.md`
* `cymatic_pulse_sound_boris.mp3`

## 🅅 Submission Workflow · Einreichung

**EN:**

1. Fork or clone the repository
2. Create a new branch for your contribution
3. Add your files in the correct folder
4. Submit a **Pull Request** with a short summary
5. Tag your contact or field (e.g. “Builder: Visual Geometry”)

**DE:**

1. Fork oder klone das Repository
2. Erstelle einen neuen Branch für deinen Beitrag
3. Lege deine Dateien im passenden Ordner ab
4. Reiche einen **Pull Request** mit kurzer Beschreibung ein
5. Füge deinen Kontakt oder Bereich hinzu (z. B. „Builder: Visual Geometry“)

## 🅆 Resonance Etiquette · Resonanz-Kultur

**EN:**

* Respect all contributions — each is a frequency.
* Cite your sources and collaborators.
* Never overwrite someone’s work; expand it.
* Keep visuals and texts lightweight and open.
* Harmony > hierarchy.

**DE:**

* Respektiere alle Beiträge – jeder ist eine Frequenz.
* Quellen und Mitwirkende immer nennen.
* Bestehende Werke nicht überschreiben, sondern erweitern.
* Visuals und Texte möglichst leicht und offen halten.
* Harmonie > Hierarchie.
